# AI 讲解后台并发 ExecPlan

## 目标

让每张卡片的 AI 讲解成为独立后台任务。用户切换卡片、收起讲解面板或继续复习时，已开始的讲解继续生成；其他卡片可以同时开始生成。已保存讲解仍可按当前思考档位重新生成，并且只在新结果完整成功后覆盖旧内容。

## 边界

- 只调整卡片讲解、错题本行内讲解及对应面板状态。
- 主 AI 助教的翻译判分、写作批改等请求继续使用原来的单任务停止逻辑。
- 不改变模型、网关地址、Key、提示词和 Low / High / Max 默认值。
- 不持久化未完成流；刷新页面会结束浏览器中的后台任务，但已完整保存的讲解不受影响。

## 设计

- 使用 `cardKey` 管理独立运行时任务，每个任务拥有自己的 `AbortController`、重试计时器、阶段、文本缓冲和所选路由。
- 流式任务只在对应卡片面板挂载时绘制；面板不存在时继续接收并缓冲。
- 重新打开同一卡片时恢复当前进度；成功后写入既有 `mti_ai_explain` 缓存并触发云同步。
- 关闭按钮只收起面板，不停止后台任务；面板提供独立的“停止此任务”入口。
- 重新生成期间保留旧缓存；新结果成功后原子覆盖，失败或停止时旧结果继续可用。

## 验收条件

1. 卡片 A 开始生成后切到卡片 B，B 可以立即开始生成，A 不被取消。
2. 回到 A 可看到进行中的流或完整已保存结果。
3. 停止 A 不影响 B；关闭任一面板不停止任务。
4. 已保存讲解可以重新生成；新结果成功后覆盖，失败或停止时保留旧结果。
5. 流式更新不抢用户滚动位置，桌面与 390px 无横向溢出或控件遮挡。

# 判分解析云端永久保存 ExecPlan

## 目标

把 211 词汇、357 词条和 448 名词解释的 AI 判分解析从“仅本机最近 180 条”升级为 Supabase 云端永久保存。保留本机缓存作为秒开和断网兜底，换设备或清除浏览器数据后，打开同一道题仍可从云端恢复。

## 边界

- 沿用项目现有 Supabase 项目，新建专用 `ai_judge_records` 表；不改模型、网关、Key、提示词、判分与 SRS 规则。
- 云端按题目独立记录，覆盖重判只覆盖该题的最新解析；失败或停止不覆盖旧解析。
- 本机仍只保留最近 180 条，云端记录不因本机 LRU 淘汰而删除。
- Supabase 离线时先保存在本机并自动重试；恢复解析不得重复学习统计或推进任务。

## 设计

- 每道题使用 `SB_ID + '_judge_' + 编码后的题目 key` 作为 `ai_judge_records.id`，独立保存题目 key、来源签名与完整判分记录。
- 数据库触发器拒绝较旧 `recorded_at` 覆盖较新记录，避免离线旧设备晚上传时倒退云端版本。
- 新解析先原子写入本机，再进入云端队列；队列批量 upsert，失败后保留并退避重试。
- 每次打开题目都按精确云端 id 核对最新版本：云端较新则更新本机，离线本机较新则补传云端。
- 首次启动把现有本机 180 条迁移到云端，不需要用户重新判分。
- 状态栏区分“本机已保存、同步中、永久保存成功、暂未同步”。

## 验收条件

1. 新解析成功后明确显示“已永久保存到云端”。
2. 删除该题本机缓存或换独立浏览器状态后，打开同题可从云端恢复输入、判定和完整解析。
3. 覆盖重判成功后云端只保留最新解析；失败、停止、无效判定不覆盖旧解析。
4. Supabase 暂时不可用时本机结果不丢，恢复联网后自动补传。
5. 云端恢复不触发 `tdOnJudged`、`grade` 或学习反馈动画。
